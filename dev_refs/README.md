# dev_refs/ 参考文件夹说明

> **用途**: 专门存放所有功能和流程的 API 接口相关代码规范，
> 方便后续开发时 AI 自动阅读此文件夹，不乱用 API，懂得借鉴已有代码。

## 文件索引

| 文件 | 内容 | 适用场景 |
|------|------|----------|
| `01-config-system.md` | 配置系统：config.json 结构、API Key 管理、路径常量、日志 | 任何需要读/写配置时 |
| `02-llm-calling.md` | LLM 调用规范：3种调用方式、多级降级、注意事项 | 任何需要调用 AI 时 |
| `03-bilibili-api.md` | B站 API：WBI 签名、BiliClient、字幕获取、Cookie | 任何需要调用 B站 API 时 |
| `04-web-panel-api.md` | Web 面板 API：Flask 路由、JS 前端模板、页面结构 | 新增 Web 面板功能时 |
| `05-web-search.md` | 网络搜索：多引擎搜索、AI 验证、结果格式 | 需要联网搜索时 |
| `06-file-io.md` | 文件 I/O：JsonStore、原子写入、路径管理、命名规范 | 需要读写文件时 |
| `07-service-template.md` | 新服务模块模板：完整代码骨架、CLI/Web 集成步骤 | 新增功能模块时 |
| `08-architecture-flows.md` | 架构与流程：项目结构、数据流、设计模式 | 了解整体架构时 |

## 使用方式

### 对 AI
在开发新功能时，AI 应该：
1. 先读 `08-architecture-flows.md` 了解整体架构
2. 根据需要读对应的参考文件
3. 使用 `07-service-template.md` 的模板创建新服务
4. 遵循各文件中的代码模式和注意事项

### 对开发者
1. 新增功能后，如果引入了新的 API 调用模式或流程，更新对应的参考文件
2. 如果发现现有参考文件中的信息过时，及时修正
3. 保持这些文件简洁、准确、实用

## ⚠️ 更新规则

- 每次新增/修改 API 调用模式 → 更新对应参考文件
- 每次新增服务模块 → 检查是否需要更新模板
- 保持与 `DEV_REFERENCE.txt` 的一致性
- 这些文件供 AI 自动阅读，代码片段必须可直接复用
